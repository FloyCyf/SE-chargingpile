#### #1 + #2: `simulate_battery_growth` + `_finish_charging`

```python
# 关键: 增量按"实际虚拟时间差"计算, 不是 (tick * ratio), 避免 tick
# 被锁/DB 拖慢时累积漂移导致车辆延迟完成.
elapsed_min = (current_vtime - last_tick_vtime).total_seconds() / 60.0
last_tick_vtime = current_vtime
increment = pile.power * elapsed_min / 60.0
car["charged_kwh"] += increment

# _finish_charging 内: cancel/stop/fault 触发时按时间精确补算
elapsed_hours = (current_vtime - charge_start_in_mem).total_seconds() / 3600.0
precise_kwh = elapsed_hours * pile.power
actual_kwh = min(max(precise_kwh, tick_kwh),
                 finished_car["requested_kwh"],
                 max_chargeable)
```

## 二、Bug 调查溯源

### 第一次修复后(只改 tick 0.2s + 精确补算 + FIFO submit): 12/22 PASS

意外:V12/V13/V16/V17/V19/V22 全部 WAITING,V14 显示 FAULTED 但实际 75kWh full。

调查 → 发现 `scheduler.py` 新增的日志含 `✓` `→` 等 Unicode 字符,Windows 控制台默认 GBK 不能编码,**后台 dispatch_watcher 第一次打印就崩溃静默退出**,导致 7 分钟后所有等候区车辆滞留。

### 第二次修复后(GBK + UTF-8): 17/22 PASS

剩余失败:V6(3.08/5)、V8(+1.29)、V11(34/35)、V14(FAULTED)、V17(-1.9)。

调查 → 用 `server.log` 抓取后端实际行为,发现**关键日志**:

```
[Clock 10:11:45] 车辆 V1 于 T1 充满 40.00度 出场。
```

V1 本应 10:00 完成,实际**晚了 11 分 45 秒**!根因:

- 旧 tick 公式 `increment = power × (ratio × 0.2) / 60` 假设每次 sleep 精确 0.2s
- 真实 `asyncio.sleep(0.2)` + 锁等待 + DB 写实际平均 ~0.21s
- 5% 漂移 × 4 小时 = 12 分钟延迟
- V1 晚完成 → V6 promote 时间 = 10:11:45 → V6 只充 18 分钟到 10:30 故障 = 3.08 kWh ❌

### 第三次修复(elapsed-based tick): **22/22 PASS** ✅

把固定增量改成按 `current_vtime - last_tick_vtime` 算实际经过的虚拟分钟数,自适应消除漂移。

## 四、给团队的提醒

### 1. UTF-8 编码强制是 Windows 部署必需

`uvicorn.exe` 在 Windows 下默认继承 GBK 控制台编码,任何 emoji/箭头/数学符号都会导致后台任务崩溃。生产部署必须:

```bash
# Windows
set PYTHONIOENCODING=utf-8
uvicorn src.main:app

# 或 systemd / Docker:
Environment="PYTHONIOENCODING=utf-8"
```

### 2. 实时模拟引擎避免固定步长假设

任何形如 `value += rate × fixed_dt` 的累计计算都应改为 `value += rate × actual_dt`,后者用墙钟差计算。否则任何抢锁/IO/调度延迟都会累积漂移。

### 3. FIFO 不能由 submit 越过

`submit_request` 直接尝试 optimal_pile 的优化会让"刚到的车插队",违反"叫号最先一辆"的需求语义。规则补丁:`len(同类型 waiting) == 0` 才允许直接进桩。

### 4. 调度日志是定位疑难杂症的关键

后续如果再遇到调度异常,启用 `g8_test.py` 的 `server.log` 抓取(已默认开启),然后 `grep -E "Dispatch|车辆"` 看时间轴。
