## 四、3 处失败的根因分析

### 失败 1: V11 偏差 1 kWh (-1.50 元)

- **场景**: V11 在 F1 充电 08:10-09:20(70 分钟 @ 30kW = 35 kWh)
- **后端实际**: 34 kWh
- **根因**: `simulate_battery_growth` 每真实秒推进一次,`increment = power × ratio / 60`(分钟数)。ratio=4 时每秒推进 4 虚拟分钟。70 虚拟分钟需 17.5 个 tick;若取消事件在 tick 中间触发会少算一个 tick(=4 虚拟分钟 = 2 kWh 偏差,实测为 1 kWh)
- **归属**: 🟠 **后端实现 → 精度问题**, 非用例错也非脚本错
- **建议**: 降低 tick 粒度(每 0.1s 或事件驱动);或在取消瞬间精确补算 `(now - tick_last) * power`

### 失败 2: V20 偏差 1.5 kWh (-2.25 元)

- **场景**: V20 在 F1 充电 09:20-09:35(15 分钟 @ 30kW = 7.5 kWh)
- **后端实际**: 6 kWh
- **根因**: 同上, tick 粒度问题。15 虚拟分钟仅 3.75 tick,误差更明显
- **归属**: 🟠 **后端实现 → 精度问题**

### 失败 3: V22 状态完全错位

- **场景**: V22 10:00 慢充提交 → 10:20 改快充
- **预期**: V22 修改成功 → F1 充电 → 10:20-10:40 完成 10kWh
- **实际**: 修改失败 → 回退取消 → CANCELLED 0kWh

后端日志:
```
[10:20] 修改 V22 失败: 车辆已在充电区，不允许修改，请先取消
```

- **根因**: V22 在 10:00 提交后,后端 `dispatch_watcher`(2 秒轮询)在 10:20 之前把 V22 调度进了某个 慢充 桩队列。具体可能链:
  1. 10:00 V22 提交,T1=[V6,V10,V16] / T2=[V5(charging),V12,V13] 双桩满,V22 → 等候区慢
  2. 10:25 V5 完成 → V12 升位 0 充电 → T2 出空位
  3. dispatch_watcher 触发 → 按 FIFO 把 V17 进 T2 pos 2(我模拟器同结论)
  4. 但实测 V22 进了 T2 → 说明 V17 没在 V22 前先调度 = **可能 dispatch_watcher 排序不严格遵循 FIFO**;或 V19 modify 10:05 触发了某次调度让 V22 提前进桩
- **归属**: 🟠 **后端实现 → dispatch_watcher 行为与 FIFO 不完全一致**
- **建议**: 检查 `_dispatch_from_waiting_area()` 的实际顺序,确认 `waiting_list` 是按提交时间 FIFO 还是按 priority_score 排序;若混入了 priority,会让 V22 提前

## 五、修正后未被发现的潜在隐患(代码侧)

### 隐患 A: scheduler.py 同时存在两套调度逻辑

- 实际调度路径(正确): `_find_optimal_pile()` 按最短完成时间选桩 → 符合需求
- 残留路径(误导): `calculate_priority()` α/β/γ 公式 + `_sorted_by_priority()` 用于展示
- 风险: 答辩/验收时评委看到管理端"按优先级降序"会误判调度策略
- 建议: ① 删除 calculate_priority 公式,展示也按提交时间 FIFO;② 或加 UI 注释"此排序仅展示,实际调度按 FIFO + 最短完成时间"

### 隐患 B: simulate_battery_growth tick 粒度

- 当前每 1 真实秒推进一次;高 ratio 下每 tick 跨度大,造成 V11/V20 偏差
- 建议: 缩短到每 0.2-0.5 秒;或在 cancel/fault 触发瞬间用 `(now - last_tick) * power` 精确补算

### 隐患 C: dispatch_watcher 与 FIFO

- 若 `_dispatch_from_waiting_area` 用了 priority 排序(`_sorted_by_priority`)代替 FIFO,会让晚到的车插队
- 实际代码第 593 行 `sorted_cars = list(waiting_list)` 并未排序 ✓
- 但 V22 现象暗示实际行为偏移,需追查
