# 人工验收宽松测试用例

本文档是一套替代 G8 的人工演示用例。目标不是塞满所有车辆，而是在虚拟时间间隔足够大的前提下，覆盖排队、取消、跨时段计费、故障持续时间、故障优先级调度和自动恢复。

## 一、适用场景

G8 用例事件间隔较密，如果按现实时间手动逐条输入，容易出现输入延迟，导致虚拟世界中的开始时间、结束时间、账单详单与预期不一致。

本用例做了两点调整：

1. 在老师要求的 1:10 比例尺内，把虚拟事件间隔尽量拉大到 30 到 45 分钟，现实中每步约有 3 到 4.5 分钟操作窗口。
2. 每辆车数量较少，主要验证逻辑，不追求大规模压测。

## 二、管理端时间设置

建议管理端这样设置：

| 项目 | 设置值 |
|---|---|
| 允许开始时间 | `2026-06-01 06:00` |
| 允许结束时间 | `2026-06-01 11:00` |
| 真实时间 | `30` 分钟 |
| 虚拟时间 | `300` 分钟 |
| 自动倍率 | `0.166667`，即真实 1 秒 = 虚拟 1/6 分钟 |

换算关系：

```text
真实 1 分钟 = 虚拟 10 分钟
虚拟 30 分钟 = 真实 3 分钟
虚拟 45 分钟 = 真实 4 分 30 秒
```

这样仍然满足老师要求的“比例尺 1:10，三十分钟对应三百分钟，共计 5 小时，起始输入时间 6:00，结束输入时间 11:00”，同时比 G8 的密集输入更容易人工操作。

## 三、操作原则

每条车辆事件都按以下方式执行：

1. 打开一个客户端页面，或复用同一个客户端页面。
2. 输入车牌号。
3. 点击“注册车辆”，如果已注册则相当于登录。
4. 设置“本车虚拟开始时间”为事件时间。
5. 选择快充或慢充。
6. 输入请求电量。
7. 提交充电请求。

故障事件在管理端执行：

1. 将管理端“设置时间”或当前虚拟时间调整到故障事件时间。
2. 在故障持续时间中填写对应分钟数。
3. 点击指定充电桩的“设为故障”。

## 五、预期关键结果

### 1. V208 取消不出账

V208 在 `09:30` 提交慢充请求，`10:00` 取消。由于它设计为排队车辆，理想状态下取消时仍未开始实际充电，因此：

```text
状态：CANCELLED
已充电量：0
账单：不生成
详单：不生成
原因：取消时未开始充电
```

### 2. V201 产生跨谷/平详单

V201 预计从 `06:00` 开始慢充 30kWh，慢充功率 10kW，理论用时 3 小时，结束时间约 `09:00`。

计费时段：

```text
06:00-07:00 谷时
07:00-09:00 平时
```

理论费用：

```text
谷时电量：10kWh，电费 10 * 0.4 = 4.00
平时电量：20kWh，电费 20 * 0.7 = 14.00
服务费：30 * 0.8 = 24.00
总计：42.00
```

### 3. V206 产生跨平/峰详单

V206 预计从 `08:30` 开始快充 60kWh，快充功率 30kW，理论用时 2 小时，结束时间约 `10:30`。

计费时段：

```text
08:30-10:00 平时
10:00-10:30 峰时
```

理论费用：

```text
平时电量：45kWh，电费 45 * 0.7 = 31.50
峰时电量：15kWh，电费 15 * 1.0 = 15.00
服务费：60 * 0.8 = 48.00
总计：94.50
```

### 4. F1 故障优先级调度

`10:15` 对 F1 设置故障 45 分钟后，验收重点不是某辆车一定落在哪个桩，而是调度规则：

```text
F1 故障后，F1 原队列车辆进入故障优先队列。
普通等待区暂停叫号。
其它快充桩出现空位时，先调度 F1 故障队列车辆。
故障队列清空后，普通等待区才恢复叫号。
```

因此 `10:30` 新来的 V209 不应抢在 F1 故障队列车辆前面。

### 5. F1 自动恢复

故障持续时间为 45 分钟，所以：

```text
故障开始：10:15
理论恢复：11:00
```

到 `11:00` 后，管理端应看到 F1 状态从“故障中”恢复为空闲或可服务状态。

## 六、建议观察页面

管理端重点看：

1. 充电桩状态：F1/F2/F3/T1/T2 的当前车辆和状态。
2. 验收快照：车辆所在区域、已充电量、当前费用。
3. 等待区：普通等待区是否暂停/恢复。
4. 故障日志：F1 故障和自动恢复记录。
5. 订单/账单/详单导出。

客户端重点看：

1. 当前车辆是否必须先注册/登录。
2. V107 取消后是否不出账。
3. V201、V206 的账单和详单是否按虚拟世界时间分段。
4. 充电完成弹窗中的结束时间是否为虚拟时间。

## 七、和 G8 的区别

本用例不替代正式表格，只用于系统演示和人工验收排查。

相比 G8：

1. 车辆数量更少。
2. 虚拟时间跨度更大。
3. 操作窗口更宽松。
4. 更适合人工逐条输入。
5. 重点验证系统逻辑，而不是精确复刻密集事件流。
