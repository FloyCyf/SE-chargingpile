## 🚀 一、 答辩演示：环境备战与走位
正式登台或者给老师验货前，你的电脑屏幕应该这样布置：

1. **后台心跳起搏**：
   在 VSCode 的终端下输入指令启动引流：
   ```bash
   uvicorn src.main:app --reload
   ```
2. **左右分屏拉满（全场焦点所在）**：
   - 屏幕**左半边**：用浏览器打开 `http://127.0.0.1:8000/admin.html`（这是给评委看的**“全局调度监控枢纽中心 Admin 大屏”**）。
   - 屏幕**右半边**：用浏览器打开 `http://127.0.0.1:8000/index.html`（这是给车主实操的**“客户端呼叫终端”**）。

> 到这里，你仅仅是铺开了这两个极具赛博科技感与毛玻璃渐变特效的界面，老师和台下的同学就知道你们这组肯定是不及格不了的。

## 📸 二、 PPT 必放截图与实况演示环节（高分脚本）

为了把所有的系统能力向不懂技术的人包装推销出去，请严格按照以下 5 个递进的环节截取 PPT 素材并向评委演说：

### 🎬 亮点 1：展示自研“无框架全栈代理层”
* **你要这么说**：“我们小组没有使用老旧的 Swagger 手工调接口，由于考虑到未来系统的轻量私有化部署，我们放弃了沉重的 React/Vue 技术栈。采用了纯原生 HTML5 搭配原生 Fetch 钩子，由后端的 FastAPI 直接承担极速代理，实现了微秒级的防跨域双向渲染。”
* **截什么图**：将左右两块大屏的**默认初始态**截下，作为 PPT 的主视觉。如果有人质疑你的花里胡哨，打开 `src/frontend/admin.html` 顺手把不到 200 行零框架依赖的纯手工结构亮一下。

### 🎬 亮点 2：一键智能寻轨上桩（展示 FIFO 派单）
* **你要做的事**：当着老师的面，在右边悬浮窗口的表单里，输入一个车牌（比如 `京A·88888`），拉选“快充”，点击**“发送并进入调度序列”**。此时请全员立刻看左侧的 Admin 大屏。
* **你要这么说**：“大家可以看到，当第一辆车进入时，左侧 Admin 控制台中，编号为 `F1` 的坑位瞬间被点亮为绿色。这就是底层的 FIFO 调度策略在零延迟下发挥了作用。”
* **截什么图**：右侧客户端回执打出**绿色通过**字样 + 左侧某个卡片被**🟢 绿色高亮**唤醒并绑定了车牌名字的画面。

### 🎬 亮点 3：虚拟时钟下的火力全开（展示电池沙盒）
* **你要做的事**：不要停，在右边接着按三次申请，把左边屏幕上的 3 个快充桩全部点亮！
* **你要这么说**：“老师请看，我们的系统并不是在简单记录数字。我们在底层挂载了一个不受 I/O 阻塞的**虚拟时钟推演沙盒**！在系统中，现实世界的 1 秒等同于 1 虚拟分钟，那些正在充能桩上的车辆，其对应的 SOC 进度条正在进行高密度地爬升，且完全没有阻塞住服务器接受下一个请求。满了以后，它们会自动结算出场！”
* **截什么图**：左侧 3-4 个坑位全部在亮着绿灯疯跑进度条（最好截取进度在 50% - 90% 的瞬间）。

### 🎬 亮点 4：触发防爆盾（展示队列溢出熔断）
* **你要做的事**：在左侧快充慢充坑位被全部占满、并且长轮询上方的大字显示“滞留数”已经达到满载时。在右侧疯狂且快速地点击请求。
* **你要这么说**：“真实电站会遭遇短时的高峰脉冲。当我们把队伍拉满以后，各位可以清晰地在右面看到，系统发起了无情地熔断拒载机制（红字拦截拦截了握手），不给后端数据库留下任何一条脏数据！”
* **截什么图**：右边黑框控制台里弹出**红色高亮**标语：“拒载：等候区已满”的报错反馈！

### 🎬 亮点 5：不浮于表面的硬核底盘（揭开 SQL 落库）
* **你要做的事**：用任意的 VSCode 插件（如 SQLite Viewer）在左侧菜单栏打开根目录下的 `charging.db`，点进 `charge_order` 表。
* **你要这么说**：“虽然前端看着炫酷，但我们并不是‘金玉其外’。刚才系统里发生的每一次拔插、每一次排队变位，均已经按照微服务级别要求通过异步事务加锁，全量登记进这个物理硬盘上的 SQLite 表里面。这为我们在后续【增量三】马上要切入账单计费的报表打好了最坚实的后路保证！”
* **截什么图**：展示布满了时间戳的 `charge_order` 数据库列表。
