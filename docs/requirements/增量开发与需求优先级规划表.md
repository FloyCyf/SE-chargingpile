# 智能充电桩调度计费系统 - 增量需求开发与优先级规划

## 1. 概述
为确保对系统核心价值的最大化交付，本文件在《软件需求规格说明书》的框架下，将全部需求以 **增量（Increment）** 为维度进行细粒度拆解，明确各个阶段的交付目标、详细业务需求以及每项要求的优先级（Priority）。

**优先级定义标准：**
* **P0（核心必选 / 阻塞级）**：缺少它系统完全无法跑通基本流程，必须在当前迭代 100% 兑现。
* **P1（重要主线 / 体验级）**：支撑完整业务流不可或缺的部分，若缺失会极大影响评委或客户体验，尽量在当前增量交付。
* **P2（优化边缘 / 加分项）**：系统能容忍其暂时缺失（或采用极简方案替代），主要用于产品后期的炫技验证、极端异常的平滑处理。

---

## 2. 增量 I：基础功能与计费模块
**目标**：实现由于车辆排队到结账的基础流转，包括基本界面、队列调度与计费。

| 需求 ID | 功能模块 | 详细说明 | 优先级 |
| :--- | :--- | :--- | :---: |
| REQ-I-01 | **区域与队列定义** | 实现等候区（区分为快充F、慢充T类型号码）与充电桩定长队列的数据结构实现。 | P0 |
| REQ-I-02 | **监控面板** | 实时显示5个充电桩状态（空闲/充电中）及各队列排队情况。 | P0 |
| REQ-I-03 | **基础排队调度** | 用户提交请求，生成对应号牌（如F1, T1），安排车辆进入等待队列或充电桩队列。 | P0 |
| REQ-I-04 | **虚拟时间推演** | 采用虚拟内部时钟推演时间，用于模拟计算充电时长与电池SOC进度。 | P0 |
| REQ-I-05 | **分段计费系统** | 根据峰时(1.0/度)、平时(0.7/度)、谷时(0.4/度)不同阶段结算费用，外加0.8/度的基础服务费。 | P0 |
| REQ-I-06 | **订单结算落库** | 充电完成后结算最终账单明细并存入SQLite数据库，释放此充电桩。 | P0 |

## 3. 增量 II：核心调度策略及请求管理
**目标**：实现文档要求的“最短时间调度策略”，并处理用户改单及取消请求生命周期。

| 需求 ID | 功能模块 | 详细说明 | 优先级 |
| :--- | :--- | :--- | :---: |
| REQ-II-01 | **最短完成时间调度** | 调度新车时计算 `每个可用桩前序车辆全部所需时间 + 当前车辆所需时间`，派发至总时间最少的桩。 | P0 |
| REQ-II-02 | **等候区请求修改** | 允许用户在等候区修改：改模式（快/慢）需重新生成号码并排在队尾；改电量要求则保留排队位置。 | P0 |
| REQ-II-03 | **充电区禁止修改** | 明确规定一旦车辆进入充电区（含所排的桩队列），即不可更改任何请求参数。 | P1 |
| REQ-II-04 | **主动取消规则** | 等候区、充电区均支持用户主动结束充电；处于充电中的车辆拔枪后立刻终止计费并出具账单。 | P0 |
| REQ-II-05 | **系统参数设置** | 供管理员在端侧调整：等候区容量(N)、桩队列长度(M)及充电功率(快/慢充)的实时期望值。 | P1 |

---

## 4. 增量 III：异常处理算法与多车调度
**目标**：实现充电桩设备异常报修时的补偿及重新排队分配动作，生成完整报表，以及批量调度的探索。

| 需求 ID | 功能模块 | 详细说明 | 优先级 |
| :--- | :--- | :--- | :---: |
| REQ-III-01 | **设备故障结算** | 单点设备故障后，当前正在该桩充电的车辆强制停表结束充电，并产生阶段性对应单据。 | P0 |
| REQ-III-02 | **优先级异常调度** | 桩故障时，其队列内未开始充电的车辆强制分配至同类型其余健康桩队列。此期间外围等候区暂停叫号。 | P0 |
| REQ-III-03 | **时间顺序恢复调度**| 故障桩修复后，合并此桩同类型其余桩所有仍在排队的车辆，按原来号码顺序重新打乱调度分配。 | P1 |
| REQ-III-04 | **多维数据详单与报表** | 将充电账单汇聚，向管理员展示多时间维度（日/周/月度）营运数、充电度数与计费的汇总看板。 | P1 |
| REQ-III-05 | **【加分项】多空位单次调度**| 当充电区同时面临多空位并行空出时，不考虑先后按串行进桩，求多车入桩同时达成最优时长效率。 | P2 |
| REQ-III-06 | **【加分项】满载批量调度** | 当等候区+充电区爆满时再启动发车，无视模式(快/慢)区分，批量盲插只求全局时间最快。 | P2 |
