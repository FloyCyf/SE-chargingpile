# 智能充电桩调度计费系统 - 增量需求开发与优先级规划

## 1. 概述
为确保对系统核心价值的最大化交付，本文件在《软件需求规格说明书》的框架下，将全部需求以 **增量（Increment）** 为维度进行细粒度拆解，明确各个阶段的交付目标、详细业务需求以及每项要求的优先级（Priority）。

**优先级定义标准：**
* **P0（核心必选 / 阻塞级）**：缺少它系统完全无法跑通基本流程，必须在当前迭代 100% 兑现。
* **P1（重要主线 / 体验级）**：支撑完整业务流不可或缺的部分，若缺失会极大影响评委或客户体验，尽量在当前增量交付。
* **P2（优化边缘 / 加分项）**：系统能容忍其暂时缺失（或采用极简方案替代），主要用于产品后期的炫技验证、极端异常的平滑处理。

---

## 2. 增量 I：核心框架搭建与基础流转验证 (MVP 阶段)
**目标愿景**：能跑起来！后端的异步框架搭建完毕，前端“双端界面”能跑通一条“提交请求 -> 进队排号 -> 取号充电 -> 到点结束”的基础全链路（暂不包含高级算力与复杂的时计功能）。

| 需求 ID | 功能模块 | 详细说明 | 优先级 |
| :--- | :--- | :--- | :---: |
| REQ-I-01 | **基础工程骨架** | 基于 FastAPI + SQLite 搭建后端基础脚手架与 API 规范路由。 | P0 |
| REQ-I-02 | **双端原生看板建立** | 车主用简易投递页（原生 HTML/JS）、Admin 管理员数据枢纽展示面板建立。 | P0 |
| REQ-I-03 | **基础身份系统** | 支持简略验证、根据输入记录车辆与对应请求的发起身份。 | P1 |
| REQ-I-04 | **先入先出(FIFO)调度** | 初期暂不上线高级数学公式，先让队列遵循“按时间戳先来先服务”跑通物理分配。 | P0 |
| REQ-I-05 | **模拟虚拟时钟基石** | 后台心跳守护进程建立，推进时间前进，保证后端不仅是被动 API，还能主动触发更迭。 | P0 |
| REQ-I-06 | **桩体状态有限状态机** | Admin 前端上可视化看到特定 ID 充电桩发生 `空闲 -> 占用(充电中) -> 空闲` 闭环。 | P0 |

---

## 3. 增量 II：核心调度灵魂与动态展示 (智能演进阶段)
**目标愿景**：显现智慧！剥离初级的 FIFO 调度模块，重构接入带有复合变量方程的评分计算核心，并在前端全面展现车主等待期被“优先级洗牌”及充电时的动画体验。

| 需求 ID | 功能模块 | 详细说明 | 优先级 |
| :--- | :--- | :--- | :---: |
| REQ-II-01 | **动态优先级公式引擎** | 引入 $SOC_i$、排队时间 $t$、快慢桩属性三项变量及管理员系数，计算动态分数 $P$。 | P0 |
| REQ-II-02 | **五大核心调度突发事件** | 完善在入队、完成退场、提前终止、异常掉线、心跳推演 这5大物理触发场景下的唤醒机制。 | P0 |
| REQ-II-03 | **用户主动撤单功能** | 车主端增加正在排队期间的“无责取消”与充电期间“终止并结账”按钮并真实生效请求。 | P1 |
| REQ-II-04 | **精细化排队插队动效** | 管理台或车主排队页应能动态折射前面有多少车，或因后车权重极高导致的排名滑落。 | P1 |
| REQ-II-05 | **充电进度条仿真** | 车主端能看到基于当前充电时间算出的 SOC 动态累进效果，预示大概多少分钟能结束。 | P1 |
| REQ-II-06 | **设备异常故障演练** | 用于供教师验收时手动在 Admin 将某个桩标记“损坏离线”，考验系统转移车辆逻辑能力。 | P2 |

---

## 4. 增量 III：分时计费商业闭环与极客管理面板 (完备交付阶段)
**目标愿景**：算明账目！不仅能调，还能准确开出包含分时阶梯电费和滞纳金在内的复合账单，并把报表管理等企业级功能投射到管理员大屏上。

| 需求 ID | 功能模块 | 详细说明 | 优先级 |
| :--- | :--- | :--- | :---: |
| REQ-III-01 | **天然区间切片分时电费** | 根据充入的时间落到的（峰、平、谷）三个区域，分别对应 `1.0 / 0.7 / 0.4` 切片乘算金额。 | P0 |
| REQ-III-02 | **充电基础服务费核算** | 常平态 `0.80 元/度`，伴随切片电量一起生成。 | P0 |
| REQ-III-03 | **三十秒级逾期滞纳金收缴** | 基于车辆落闸结余缓冲逾期后的每分钟单价计入最终应结总额中。 | P1 |
| REQ-III-04 | **数据留底查询与电子账单** | 提供带订单号、两端 SOC、开始结束时间戳的详单下发逻辑。 | P0 |
| REQ-III-05 | **Admin 高级运维面板全貌** | 呈现账单与营业额的基础图表流水。 | P1 |
| REQ-III-06 | **核心系数热部署** | 允许在管理看板对优先级公式内的系数和等候区容量进行即时重配置。 | P2 |
| REQ-III-07 | **WebSocket 消息实时推送** | 取代传统的 API 请求刷新，彻底改造车主端由服务端 WS 实时推送状态变更弹窗通知。 | P2 |
