# 智能充电桩调度计费系统 - 增量需求开发与优先级规划

## 1. 概述
为确保对系统核心价值的最大化交付，本文件在《软件需求规格说明书》的框架下，将全部需求以 **增量（Increment）** 为维度进行细粒度拆解，明确各个阶段的交付目标、详细业务需求以及每项要求的优先级（Priority）。

**优先级定义标准：**
* **P0（核心必选 / 阻塞级）**：缺少它系统完全无法跑通基本流程，必须在当前迭代 100% 兑现。
* **P1（重要主线 / 体验级）**：支撑完整业务流不可或缺的部分，若缺失会极大影响评委或客户体验，尽量在当前增量交付。
* **P2（优化边缘 / 加分项）**：系统能容忍其暂时缺失（或采用极简方案替代），主要用于产品后期的炫技验证、极端异常的平滑处理。

---

## 2. 增量 I：核心基础版本 (MVP 阶段)
**目标愿景**：满足《使用充电服务》的基础跑通，核心是搭好底层架构，将业务核心难点“计费策略”与“状态流转”先建立。

| 需求 ID | 功能模块 | 详细说明 | 优先级 |
| :--- | :--- | :--- | :---: |
| REQ-I-01 | **基础工程骨架** | 基于 FastAPI + SQLite 搭建后端基础脚手架与 API 规范路由。 | P0 |
| REQ-I-02 | **五桩监控Admin面板** | [管理员] UC2.1 建立监控枢纽，实现对5个充电桩当前工作状态（空闲/充电中）的实时独立监控。 | P0 |
| REQ-I-03 | **特定触发条件直接入桩(事件1)** | [车] UC1.1 仅限有空闲桩时直接入桩，实现资源的即刻分配。暂不用考虑排队问题。 | P0 |
| REQ-I-04 | **模拟虚拟时钟驱动** | 后台心跳进程推进时间起手，建立时标，驱动后续按自然时间的切片核算。 | P0 |
| REQ-I-05 | **结束与自动释放(事件2)** | [车] UC1.1 充电达成目标SOC时自动落闸停机，释放物理坑位。（注：依据新规，释放占位后不自动横向触发当前队列第一顺位调度）。 | P0 |
| REQ-I-06 | **时限切片阶梯计费策略** | [车] UC1.1 严格按分段切片：波峰(10-15、18-21点)1.00元/度；波平(7-10、15-18、21-23点)0.70元/度；波谷(23-次日7点)0.40元/度。**这是MVP核心。** | P0 |
| REQ-I-07 | **基础服务费核算** | [车] UC1.1 全天候固定费率 0.80元/度，附加于每度耗电之上，在结账时统筹。 | P0 |

## 3. 增量 II：排队调度灵魂与动态展示 (智能演进阶段)
**目标愿景**：补齐排队流程与调度引擎！剥离简单模式，接入包含等候区队列与复合变量方程的评分计算核心，并展现排队动态。

| 需求 ID | 功能模块 | 详细说明 | 优先级 |
| :--- | :--- | :--- | :---: |
| REQ-II-01 | **入队排序分发(事件1)** | [车] 新请求入队不直接分配，对全队列重新排序后若有空闲则指派。进入等候区可改模式和电量，进入充电区不可更改。 | P0 |
| REQ-II-02 | **动态优先级公式引擎** | 引入 $SOC_i$、等待时间等权重参数。*(注：按照新规已废弃依据系统心跳定期推演加权分的机制，避免反超排位过分扰动)*。 | P0 |
| REQ-II-03 | **主动中止事件落地(事件3)** | [车] 支持车主中途主动拔枪终止。系统立即回收资源、按实际收钱落闸，并无缝衔接开展新一轮调度。 | P0 |
| REQ-II-04 | **故障异常补偿调度(事件4)** | [车] 硬件强制异常时原车挂起，有空闲则横向调度，无空闲由最高优先级候补。**故障充电时长不计费**。 | P1 |
| REQ-II-05 | **精细化视效与进度仿真** | 车主端能看到排队等候的排名或因让位造成的掉落，以及可视化的进度条预估完成分秒数。 | P1 |
| REQ-II-06 | **设备异常故障断联演练** | 供验收时在 Admin 标记某充电桩“损坏”，演练异常事件触发与补偿机制。 | P2 |

## 4. 增量 III：极客管理面板与企业级功能闭环 (完备交付阶段)
**目标愿景**：系统级完善！算清逾期违约账目，增强后台报表全景，确保整个项目以工业级水平完备交付。

| 需求 ID | 功能模块 | 详细说明 | 优先级 |
| :--- | :--- | :--- | :---: |
| REQ-III-01 | **超时占用惩罚金收缴** | 针对拔枪落闸缓冲：提供 **30分钟豁免期**，之后按 **1.00元/分钟极速叠加逆向惩罚**加注到最后账单。 | P1 |
| REQ-III-02 | **数据留底与全貌电子详单** | 单向提供带完整详单记录查询功能给客户端，以及全局订单存储供查询。 | P0 |
| REQ-III-03 | **Admin 高级系统全域概览** | 呈现营业额统计图表与针对排队容量、各个桩工作历史周期的总全局面貌。 | P1 |
| REQ-III-04 | **核心系数即时热部署** | 允许在管理看板任意配置优先级公式内系数、阈值。并能即使生效重构全局调度分数。 | P2 |
| REQ-III-05 | **WebSocket 状态推送流** | （备选进阶）将轮询式的轮询更迭代为基于长连接的自动推送告警。 | P2 |
