# G8测试用例系统验收操作手册

本文档用于在当前系统上验证 `D:\大三下作业\SE-chargingpile\G8测试用例.xlsx`。操作重点是按表格事件顺序录入车辆请求、取消/修改请求、制造故障、观察故障优先级调度、导出账单和详单。

## 一、是否建议补登录界面

建议补，但不建议阻塞本次 G8 验收。

当前后端已经有认证能力，管理端接口也已经通过 `Authorization: Bearer <token>` 做管理员权限校验；默认管理员账号为 `admin`，密码为 `admin123`。问题在于管理端页面还没有明显的登录入口，客户端页面也没有完整的用户登录/隔离体验，所以验收时需要先在浏览器控制台写入 token。

后续更好的方案是：

1. 管理端增加简单登录页或登录弹窗，登录成功后写入 `localStorage.admin_token`。
2. 客户端增加用户注册/登录入口，登录成功后写入 `localStorage.user_token`。
3. 后端继续保持管理员接口只允许 admin 访问；用户侧订单、车辆、账单接口按用户身份隔离。
4. 兼容接口可保留用于演示，但正式验收推荐逐步切换到带登录态的用户接口。

## 二、启动后端和初始化数据库

在 PowerShell 中进入项目目录：

```powershell
cd D:\大三下作业\SE-chargingpile
```

激活虚拟环境：

```powershell
.\.venv\Scripts\activate
```

安装依赖：

```powershell
pip install -r requirements.txt
```

初始化数据库并写入管理员和充电桩基础数据：

```powershell
python -m scripts.init_db --reset
```

注意：`--reset` 会删除当前 `charging.db` 并重建，适合验收前清空历史数据。如果要保留已有数据，不要加 `--reset`。

启动后端：

```powershell
python -m uvicorn src.main:app --host 127.0.0.1 --port 8000
```

启动后不要关闭这个 PowerShell 窗口。

## 三、打开页面

1. 管理端：`http://127.0.0.1:8000/admin.html`
2. 客户端：`http://127.0.0.1:8000/index.html`
3. 接口文档：`http://127.0.0.1:8000/docs`

如果管理端报 `401 Unauthorized`，说明还没有管理员 token。

## 四、管理端写入管理员 token

打开管理端页面后，按 `F12` 打开浏览器控制台。如果浏览器提示不要粘贴代码，需要先输入：

```js
allow pasting
```

然后执行：

```js
fetch('/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username: 'admin', password: 'admin123' })
})
.then(r => r.json())
.then(d => {
  localStorage.setItem('admin_token', d.access_token);
  location.reload();
});
```

刷新后，管理端的时钟、充电桩控制、等待区、验收快照、订单账单导出应能正常加载。

## 五、验收前基础设置

建议有两种执行方式：

1. 正式验收计时方式：按验收要求使用 1:10 比例尺，真实 30 分钟对应虚拟 300 分钟，即虚拟时间从 06:00 推进到 11:00。
2. 调试复盘方式：每执行一条表格事件前，在管理端“虚拟时间控制”中手动把时间设置到对应事件时间，适合排查单步调度问题。

当前系统中的“时间倍率”含义是：每真实 1 秒推进多少虚拟分钟。因此验收要求的 1:10 不能填 10，而应换算为：

```text
真实 1 分钟 = 虚拟 10 分钟
真实 1 秒 = 虚拟 10 秒 = 1/6 虚拟分钟
时间倍率 = 0.1667
```

建议初始设置：

1. 虚拟时间：`2026-01-01 06:00`
2. 正式验收计时方式下，在管理端填写“真实时间 30 分钟、虚拟时间 300 分钟”，系统会自动换算底层倍率为约 `0.1667`。
3. 管理端允许开始时间建议设为 `2026-01-01 06:00`，允许结束时间建议设为 `2026-01-01 11:00`。保存后，“设置时间”会自动同步为允许开始时间，“时间倍率”会根据真实时间和虚拟时间自动计算。
4. 调试复盘方式下，可以逐条事件在用户端设置“本车虚拟开始时间”，再提交请求。
5. 确认 F1、F2、F3、T1、T2 都为空闲。
6. 确认等待区为空。
7. 确认订单、账单、详单没有历史数据。

## 六、车辆注册/登录要求

用户端现在必须先注册或登录车辆，才能提交充电请求。

1. 在客户端输入车牌号，例如 `V1`。
2. 输入电池最大容量。
3. 点击“注册车辆”。
4. 如果该车牌从未注册，会创建车辆。
5. 如果该车牌已经注册，本次点击相当于登录，并会返回该车辆当前状态、所在区域、订单状态和已生成账单/详单。
6. 同一车牌同一时间只能被一个浏览器页面登录。若另一个页面已经登录该车牌，当前页面会提示该车牌已在其它页面登录。

验收时建议每个车辆使用唯一车牌号：`V1、V2、V3...`。如果要模拟多辆车，可以打开多个客户端页面，但不要在不同页面登录同一个车牌。

如果需要完全重新开始，停止后端后重新执行：

```powershell
python -m scripts.init_db --reset
python -m uvicorn src.main:app --host 127.0.0.1 --port 8000
```

## 七、表格事件含义

G8 表格中的事件可按以下方式操作：

| 表格事件 | 含义 | 系统操作 |
|---|---|---|
| `(A,Vx,T,k)` | 车辆 Vx 提交慢充 k 度请求 | 客户端先注册/登录 Vx，再选择慢充，设置本车虚拟开始时间，填写请求电量 |
| `(A,Vx,F,k)` | 车辆 Vx 提交快充 k 度请求 | 客户端先注册/登录 Vx，再选择快充，设置本车虚拟开始时间，填写请求电量 |
| `(A,Vx,O,0)` | 车辆 Vx 取消当前请求 | 客户端取消；或用接口 `/api/user/charge/cancel` |
| `(C,Vx,T/F,k)` | 车辆 Vx 修改充电请求 | 使用用户接口 `/api/user/requests/{order_id}/modify` |
| `(B,T1,O,60)` | T1 故障 60 分钟 | 管理端对 T1 设置故障，持续时间填 60 |
| `(B,F1,O,120)` | F1 故障 120 分钟 | 管理端对 F1 设置故障，持续时间填 120 |
| `Vx充电完成` | 观察点 | 不需要手动输入，观察快照、订单、账单 |

注意：当前用户页面主要支持提交和取消。若要严格执行修改请求，推荐在 `/docs` 里调用新版用户接口，或用浏览器控制台调用。修改请求需要先知道该车的 `order_id`，可在管理端订单列表或验收快照中查看。

## 九、修改请求的接口操作方式

如果页面没有修改按钮，可以先在管理端订单列表找到车辆的 `order_id`，再在 `/docs` 调用接口：

```text
POST /api/user/requests/{order_id}/modify
```

请求体示例：

```json
{
  "charge_type": "F",
  "requested_kwh": 25
}
```

该接口需要普通用户 token。可以注册一个用户后登录，也可以在 `/api/auth/register` 中创建临时用户。若只做验收演示，建议优先使用页面能直接完成的提交、取消、故障、快照和导出功能。

特别注意：如果某辆车已经进入充电桩队列或正在充电，当前后端可能拒绝修改请求。G8 中 `10:10 (C,V21,F,10)` 需要重点核对。如果系统拒绝 V21 修改，需要在验收记录中标注“当前实现不支持充电中修改请求”，或先补后端逻辑后再完整复测。

## 十、故障优先级调度观察点

本项目当前目标调度策略是故障优先级调度：

1. 充电桩故障后，故障桩队列车辆形成故障等待队列。
2. 普通等候区暂停叫号。
3. 当其它同类型充电桩队列有空位时，优先调度故障队列车辆。
4. 故障队列全部进入充电区后，普通等候区才恢复叫号。

在 G8 中重点观察：

1. `10:30` T1 故障后，T1 中受影响车辆不应直接丢失。
2. T1 故障队列未清空前，普通慢充等待区不应抢先进入 T1/T2。
3. `11:30` T1 自动恢复后，T1 状态应从“故障中”变为空闲或继续服务队列车辆。
4. `10:50` F1 故障 120 分钟后，F1 应在 `12:50` 左右自动恢复；若已有故障队列，恢复前后都应保持故障优先级调度规则。

## 十一、账单和详单核对

客户端现在也可以查看该车辆自己的账单和详单：

1. 在客户端输入车牌号并点击“注册车辆”完成登录。
2. 点击右侧账单/详单区域的“当前账单”或“历史记录”。
3. 页面会按当前登录车辆读取该车的订单、账单和分时段详单。
4. 未完成或取消车辆可能只有订单状态，没有完成账单。

完成用例后，在管理端导出：

1. 订单 CSV/Excel
2. 账单 CSV/Excel
3. 详单 CSV/Excel

导出入口对应：

```text
/api/admin/export/orders.xlsx
/api/admin/export/bills.xlsx
/api/admin/export/bill-details.xlsx
```

核对内容：

1. 已完成车辆应有账单总金额、充电费、服务费、充电电量、开始时间、结束时间。
2. 跨峰平谷时段的车辆应产生多条详单。
3. 取消车辆应标注取消原因，不应产生完成账单。
4. 未开始或未完成车辆应说明原因，不应强行填写完成账单。
5. V21、V22、V6 是本用例的重点车辆，需要重点核对修改、故障、恢复后的状态和费用。

## 十二、常见问题

1. 管理端显示 401：重新执行管理员登录 token 脚本。
2. 页面状态不刷新：点击管理端刷新按钮，或刷新浏览器页面。
3. 数据和表格对不上：先确认是否重新初始化数据库，再确认每一步是否先设置了虚拟时间。
4. 端口 8000 被占用：关闭旧的后端窗口，或改用其它端口并同步修改访问地址。
5. V21 修改失败：说明当前实现暂不支持充电中修改请求，需要补后端能力或在验收差异中记录。
6. 自动故障恢复没有发生：检查故障时是否填写了持续时间；也可以先手动恢复并记录该差异。
7. 车辆提示已在其它页面登录：关闭占用该车牌的客户端页面，或重启后端释放临时车辆会话。
