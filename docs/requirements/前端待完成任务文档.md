# 前端待完成任务文档

本文档基于《智能充电桩调度计费系统详细需求 20260327(1).docx》、当前前端页面和后端接口现状整理，面向前端开发人员。目标是把用户端和管理员端补齐为可演示、可验收、与后端契约一致的完整界面。

## 一、当前前端状态

已有页面：

- `src/frontend/index.html`：用户端页面，包含充电请求、车辆信息、状态查询、订单/账单展示等部分功能。
- `src/frontend/admin.html`：管理员端页面，包含充电桩看板、队列、报表、账单、导出、参数配置等部分功能。
- `checkout.html`：结算/订单详情展示页面。

已处理问题：

- `src/frontend/index.html` 和 `src/frontend/admin.html` 的 Git 冲突标记已清除。
- 前端已有车辆状态显示和账单导出按钮的代码雏形。

主要风险：

- 页面中仍有多处中文编码异常，需要统一 UTF-8 并修正文案。
- 前端调用了兼容接口和正式接口的混合路径，需要和后端统一。
- 等候区目前只展示排队顺序，尚未展示动态优先级分数、SOC、等待时间和重排原因。
- 虚拟时间控制、跨时段详单演示、改单流程等需求尚未形成完整用户路径。
- 管理端部分按钮仍是提示或半成品，需要改成真实接口调用。

## 二、P0 必须完成任务

### FE-P0-01 统一接口调用契约

任务内容：

- 和后端确认最终接口路径，替换页面中临时兼容路径。
- 建议集中维护 API 调用函数，避免在页面各处硬编码路径。
- 至少统一以下能力：
  - 车辆注册/查询/修改
  - 提交充电请求
  - 修改充电请求
  - 取消/结束充电
  - 查询车辆当前位置
  - 查询订单与账单
  - 管理员桩控制、报表、导出、参数设置

验收标准：

- 用户端和管理端不再出现同一功能调用两套接口。
- 后端字段变更时只需改一处 API 封装。

### FE-P0-02 完成用户注册/登录与车辆绑定入口

需求依据：

- 用户客户端需要注册、登录。
- 服务器端维护用户/车辆信息。

任务内容：

- 用户端提供注册和登录入口。
- 登录成功后保存 token 和用户信息。
- 支持绑定或注册车辆：
  - 车牌号
  - 电池总容量
  - 当前电量
- 已注册车辆再次进入页面时自动加载车辆信息。

验收标准：

- 新用户能注册、登录并绑定车辆。
- 当前电量大于电池总容量时前端提前提示，后端仍能兜底拒绝。
- 页面能展示车辆总容量、当前电量、剩余可充电量。

### FE-P0-03 完成用户充电请求主流程

需求依据：

- 用户可提交充电请求，包括快/慢充和本次请求充电量。
- 系统返回 F/T 排队号码。

任务内容：

- 请求表单包含：
  - 充电模式：快充/慢充
  - 请求充电量
  - 当前车辆
- 提交前校验：
  - 请求电量必须大于 0
  - 请求电量不能超过剩余可充电量
- 提交成功后展示：
  - 排队号码
  - 所在区域
  - 前车等待数量
  - 分配到的充电桩，若已有

验收标准：

- 快充请求显示 F 编号，慢充请求显示 T 编号。
- 成功提交后管理端看板能同步看到对应车辆。
- 请求失败时展示后端返回的明确原因。

### FE-P0-04 完成车辆当前位置展示

需求依据：

- 用户需要查看本车排队号码和本模式下前车等待数量。
- 当前项目还需要展示车辆当前在哪个区域。

任务内容：

- 调用车辆状态接口，展示：
  - 等候区
  - 充电区排队
  - 正在充电
  - 已完成/已离开
  - 故障中断
- 展示字段：
  - `area`
  - `area_detail`
  - `queue_id`
  - `queue_position`
  - `order_id`
  - 预计等待时间，若后端提供

验收标准：

- 车辆从等候区进入充电区后，页面自动刷新状态。
- 充电完成后状态变为已完成，并可查看账单。
- 状态展示不依赖手动刷新页面。

### FE-P0-05 完成动态优先级队列展示

需求依据：

- 需求规格说明书要求动态展示“由于数学权重逆转而使车辆排序变化”的反馈。
- 后端应返回 `priority_score`、`soc`、`wait_minutes` 等字段，前端需要可视化呈现。

任务内容：

- 用户端展示本车优先级相关信息：
  - 当前 SOC 或当前电量比例
  - 已等待时间
  - 优先级分数
  - 本车在同类型优先级队列中的名次
  - 前方车辆数量
- 管理端队列列表展示每辆车的：
  - 排队号
  - SOC
  - 等待时间
  - 请求电量
  - 优先级分数
  - 排序依据或重排原因
- 当优先级重算导致排序变化时，界面需要刷新并给出轻量提示，例如“队列已按优先级重排”。
- 避免把优先级队列误展示成单纯到达顺序；如果同时展示排队号和优先级名次，应明确两者含义不同。

验收标准：

- 构造低电量车辆后到但优先级更高的场景，管理端队列排序能体现反超。
- 用户端能看到自己的优先级分数和前方车辆数量变化。
- 调整管理员权重参数后，队列显示随刷新发生变化。

### FE-P0-06 完成改单、取消和结束充电入口

需求依据：

- 等候区允许修改充电模式和请求电量。
- 充电区不允许修改请求，但允许取消/结束充电。
- 等候区和充电区均允许取消。

任务内容：

- 用户端订单卡片提供：
  - 修改模式
  - 修改请求电量
  - 取消请求
  - 结束充电
- 根据订单状态控制按钮可用性：
  - 等候区：可修改、可取消
  - 充电区队列/充电中：不可修改，可取消/结束
  - 已完成：只可查看账单
- 修改成功后刷新排队号和状态。

验收标准：

- 等候区修改模式后展示新的 F/T 编号。
- 充电区点击修改时前端禁用或展示后端错误。
- 充电中结束后跳转或展示账单。

### FE-P0-07 完成用户账单和详单展示

需求依据：

- 用户可查看充电详单，至少包含详单编号、生成时间、桩编号、充电电量、充电时长、启动时间、停止时间、充电费用、服务费用、总费用。

任务内容：

- 在用户端订单历史中展示账单摘要。
- 支持点击查看详单。
- 详单中展示峰/平/谷分段：
  - 时段类型
  - 起止时间
  - 分钟数
  - 电量
  - 单价
  - 费用
- `checkout.html` 对齐同一套订单/账单接口。

验收标准：

- 充电完成后用户端能看到完整账单。
- 跨时段订单显示多条分段明细。
- 金额显示保留两位小数，电量显示合理精度。

### FE-P0-08 完成管理员充电桩监控看板

需求依据：

- 管理员需要查看所有充电桩状态、累计充电次数、累计充电总时长、累计充电总电量。
- 管理员需要查看各充电桩等候服务车辆信息。

任务内容：

- 管理端每个充电桩卡片展示：
  - 桩编号
  - 类型：快充/慢充
  - 状态：空闲/充电中/故障
  - 功率
  - 当前充电车辆
  - 队列长度
  - 累计次数、时长、电量、费用
- 队列车辆展示：
  - 用户 ID，若有
  - 车牌号
  - 车辆电池总容量
  - 请求充电量
  - 排队时长
  - 排队号

验收标准：

- 用户端提交请求后，管理员看板 2 秒内刷新。
- 故障、恢复、完成充电后状态实时变化。
- 队列车辆信息不缺关键字段。

### FE-P0-09 完成管理员桩控制

需求依据：

- 管理员可启动/关闭充电桩。

任务内容：

- 每个桩提供：
  - 启动/恢复
  - 关闭/故障
- 操作前做确认，尤其是关闭正在充电的桩。
- 操作后刷新桩状态、队列、日志。

验收标准：

- 点击故障后，桩状态变为故障。
- 如果该桩正在充电，用户端对应订单变为故障中断或已结算状态。
- 点击恢复后，同类型队列重新调度并刷新页面。

## 三、P1 重要完善任务

### FE-P1-01 完成管理员报表页面

需求依据：

- 管理端报表至少包含时间、充电桩编号、累计充电次数、累计充电时长、累计充电量、累计充电费用、累计服务费用、累计总费用。

任务内容：

- 提供日/周/月切换。
- 支持选择日期。
- 报表表格展示完整字段。
- 可视化图表只做辅助，不能替代表格。

验收标准：

- 选择日/周/月后调用正确接口。
- 报表数据与账单汇总一致。
- 无数据时展示空状态。

### FE-P1-02 完成账单、详单和导出管理

任务内容：

- 管理端账单列表支持：
  - 按车牌查询
  - 分页
  - 展开详单
- 导出按钮调用：
  - `/api/admin/export/orders`
  - `/api/admin/export/bills`
  - `/api/admin/export/bill-details`
- 下载时显示加载状态和失败提示。

验收标准：

- 三个导出按钮都能下载文件。
- 账单展开后显示详单分段。
- 导出的 CSV 与页面数据含义一致。

### FE-P1-03 增加虚拟时间控制面板

需求背景：

- 跨时段计费演示需要可控时间。

任务内容：

- 等后端提供接口后，在管理端增加：
  - 当前虚拟时间
  - 设置虚拟时间
  - 重置虚拟时间
  - 设置时间倍率
- 用户端可只展示当前虚拟时间，不允许修改。

验收标准：

- 管理端设置时间到 09:55 后，页面显示一致。
- 订单启动和结束时间使用虚拟时间。
- 可通过倍率快速完成订单。

### FE-P1-04 完成跨时段订单演示入口

任务内容：

- 在管理端提供“跨时段测试辅助区”：
  - 一键设置虚拟时间到临界点附近，例如 09:55、14:55、22:55。
  - 提示用户去用户端提交小额订单。
  - 显示最近一笔跨时段详单。

验收标准：

- 评审现场能在 1 分钟内制造出跨峰/平/谷订单。
- 页面明确展示多段费用，不需要打开数据库。

### FE-P1-05 完成管理员参数设置

任务内容：

- 对接后端参数接口：
  - 快充功率
  - 慢充功率
  - 等候区容量 N
  - 桩队列长度 M
  - 峰/平/谷电价
  - 服务费单价
  - 峰/平/谷时间段
  - 优先级权重 `alpha/beta/gamma`
  - 快充/慢充模式权重 `W_type`
- 表单中增加格式校验。

验收标准：

- 参数修改后页面刷新仍显示最新值。
- 非法时间段或负数价格会被拦截。
- 修改优先级权重后，队列优先级分数和排序能刷新展示。

## 五、前端验收清单

开发完成后至少跑通以下场景：

#1. 用户注册、登录、绑定车辆。(忽略)
2. 用户提交快充请求，显示 F 编号。
3. 用户提交慢充请求，显示 T 编号。
4. 用户查看本车位置：等候区、充电区、正在充电、已完成。
5. 等候区展示优先级分数、SOC、等待时间，并能看到优先级重排结果。
6. 等候区修改模式，编号变化；修改电量，编号不变。
7. 充电区修改请求不可用。
8. 充电中结束充电后展示账单。
9. 跨时段订单展示多条峰/平/谷详单。
10. 管理员查看所有桩状态和队列车辆。
11. 管理员关闭/恢复充电桩，页面同步故障调度结果。
12. 管理员查看日/周/月报表。
13. 管理员导出订单、账单、详单 CSV。

## 六、界面清理要求

- 所有页面统一 UTF-8 编码，修复乱码中文。
- 页面提示语与需求术语保持一致：等候区、充电区、快充、慢充、排队号、详单、账单。
- 错误提示优先展示后端返回信息。
- 按钮状态必须随订单状态变化，避免用户触发不允许的操作。
